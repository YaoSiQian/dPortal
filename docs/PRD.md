# 界门（Portal）项目说明文档

## 一、项目概述

界门（Portal）是一个基于 Web 的 3D 交互式探索平台。用户通过浏览器进入一个由真实天体和虚构星球构成的宇宙，在自由漫游、旋转与缩放中，发现散落在星球表面上的文化标记点。每个标记点以影视作品或科技成就的海报形式呈现，点击后可查看作品介绍。界门的核心体验是：将人类的文化想象与科技足迹，具象化为可探索的星球世界。

## 二、核心理念与定位

界门不做应用内的社区功能，而是通过保存探索日志、生成日志卡片、分享宇宙截图等方式，引导用户将探索成果发布至抖音等第三方平台进行互动。产品定位为个人/概念演示级别，不涉及版权授权，所有内容仅用于展示与体验。界门的目标是让用户以一种浪漫且富有沉浸感的方式，重新“看见”那些熟悉的作品与成就——它们不再是一张海报或一段文字，而是一颗星球上的一个可抵达的坐标。

## 三、星球系统

界门初始版本将展示太阳系的八大行星以及月球。这些星球不仅作为天体存在，更作为叙事容器：每一颗星球都承载着以其为故事场地的科幻作品（例如火星上标记《火星救援》与《流浪地球》中的火星场景），以及人类真实的宇宙探索科技成就（例如月球上标记阿波罗11号登月点）。未来版本计划逐步加入太阳系外真实星体（如比邻星）以及完全虚构的星球（如《星球大战》中的塔图因）。

所有星球支持自由旋转、缩放与飞行视角。用户可以用鼠标或手势从任意角度观察星球表面，拉近至海报标记点，或拉远至行星全景。

## 四、标记点系统

标记点是界门中内容的基本单元。每个标记点以影视作品的海报形式浮现在星球表面的特定位置。点击海报后，弹窗展示该作品或科技成就的详细文字介绍（包括名称、年代、简介等）。标记点允许由管理员或用户本人通过简单方式添加（例如上传包含海报图片与介绍文本的 JSON 文件），当前版本不开放线上多用户协同编辑。

在初始版本中，每个星球上预设若干标记点，涵盖以该星球为背景的科幻电影、小说、游戏，以及历史上与该星球相关的真实探索任务。随着配置文件的更新，标记点数量与内容可以动态增加，无需修改核心代码。

## 五、文化领域切换

界门内置“文化领域”切换机制。用户可在界面上选择不同的文化领域（例如科幻、二次元、武侠、古典音乐等）。切换文化领域时，星球的视觉氛围会发生变化（例如色调、光效、粒子效果等滤镜改变），同时**星球上的标记点内容也会随之替换为一套新的集合**。这意味着同一颗火星，在科幻领域下展示《火星救援》的标记点，在二次元领域下可能展示《你的名字》中虚构的火星场景，在武侠领域下则可能完全留空或展示与“火星”意象相关的诗意节点。

文化领域可以动态增加，每个领域对应一套独立的配置文件（定义星球滤镜参数与各星球上的标记点列表）。这种设计使得界门能够逐步扩展为覆盖人类多种文化兴趣的“多元宇宙”入口。

## 六、AI 导航员

界门集成了一位 AI 导航员，以浮动聊天窗口的形式与用户交互。导航员支持文字输入与语音输入（使用 Web Speech API 的 SpeechRecognition），用户可以像与导游对话一样提问，例如：“我想看关于登月的标记点。”AI 会根据当前星球与文化领域的标记点数据，推荐一条探索路径，并在 3D 场景中依次高亮或自动飞行至相关标记点。

AI 导航员不参与社区互动，也不保存对话历史，仅作为实时的探索辅助工具，帮助用户在众多标记点之间快速定位感兴趣的内容。

## 七、空间移动与跃迁动画

界门区分两种移动方式：

- **同一星球内的移动**：用户通过点击标记点或 AI 推荐的路径，视角会在星球表面平滑飞行，从一个标记点移动到另一个标记点，保持星球的完整可见性。
- **跨星球移动**：当用户从一颗星球（例如地球）切换到另一颗星球（例如火星）时，触发跃迁动画：镜头先拉远，使原星球逐渐缩小并最终消失，随后新星球在视野中央加载并拉近。整个过程不使用瞬移，而是保持镜头连续运动的沉浸感。

所有动画时长控制在 2–3 秒，避免打断探索节奏。

## 八、日志与分享

界门不提供应用内的社交功能（无评论、点赞、关注、时间线）。取而代之的是，用户可以：

- **保存当前视角**：将相机位置、缩放层级、当前星球与标记点选中状态记录为一个“书签”。
- **生成探索日志**：将本次探索过程中访问过的标记点、AI 导航员的对话记录、用户自己的文字笔记，打包生成一张日志卡片（可导出为图片或文本文件）。
- **分享截图**：一键截取当前 3D 画面，并引导用户下载或复制到系统剪贴板。

所有生成的内容均可被用户自行发布到第三方平台（如抖音、微博等）进行互动。界门本身不承载社交闭环，仅作为内容创作与探索的工具。

## 九、技术实现与交付形式

界门以 Web 端形式交付，基于 Three.js 或类似 WebGL 框架开发，兼容主流桌面浏览器（Chrome、Edge、Firefox）。不需要安装任何插件。语音识别功能依赖于 Web Speech API，在部分浏览器中可能需要用户授权麦克风权限。所有星球模型、标记点数据、AI 导航逻辑均在客户端本地运行，或通过简单的静态 JSON 配置文件加载，无需后端服务器（除非未来加入在线数据同步）。当前版本作为个人/概念演示，代码与配置可完全开源。

## 十、后续扩展方向

- 增加太阳系外真实天体（如比邻星 b）及完全虚构的星球（如《沙丘》中的厄拉科斯）。
- 支持用户通过更友好的界面（而非手写 JSON）添加私有标记点，保存至本地浏览器。
- 为 AI 导航员增加更丰富的对话能力，例如根据用户兴趣自动生成探索路线叙事。
- 引入简单的音效与背景音乐，不同星球拥有独立的环境音轨。

## 十一、结语

界门（Portal）不追求成为另一个社交平台，而是回归探索本身：一个人面对星空，点击海报，阅读一段故事，然后飞向下一颗星球。通过将文化内容空间化、可抵达化，界门希望让每一次点击都像一次着陆，每一次跃迁都像一次出发。
